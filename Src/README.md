# ソースフォルダ

この `README.md` ファイルは、このフォルダに関しての説明を記述したファイルです。

## 概要

- このフォルダは、**ソースコード**を収めるフォルダです。
- [開発フォルダ](../README.md)の下位フォルダとする事を、想定しています。
  - (フォルダ構成は「[フォルダ構成・方針](../Document/00_Materials/folder_policy.md)」で記しています。)
- ここはまとめる為の親フォルダであり、実際のソースコード等は子フォルダ(下位フォルダ)で保持します。(制御用データファイルは除く)
- 名称: **Src**, 区分: **Sr** です。

## 使い方

- 開発で新たに生成用ソースを用意する方法

    用いるツールなどタイプによって、用意するフォルダを振り分けます。各フォルダの規則にしたがって、ファイルを振り分けます。

## 内部規約

1. 主に [コーディングルール](../Document/20_Design/codingrules.md) に準じます。
2. このフォルダのフォルダ区分は、'**Sr**'(Src) です。
3. このフォルダのファイルに関する規約は、以下の通りとする。
    1. 説明ファイル, 制御用JSONファイル以外のファイルを置かない。
4. このフォルダの下位フォルダは、用いるツール別とする。
    1. ツール別であれば、１つのツールで複数のフォルダとしても良い。１つのツールで１つのフォルダに纏める必要はない。
    2. １つのツールで１つのフォルダに纏める必要がないとの判断理由は、抽出条件は拡張子で充分だと思うし、細かい条件の場合は個別指定が必要と考えるからです。また、階層をあまり深くしたくないと思うからです。
5. [基本原則](../Document/10_Principles/basicprinciples.md) にあるように、決まり事を試行錯誤する事も目的です。
