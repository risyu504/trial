# 上流資料

この `README.md` ファイルは、このフォルダについての説明を記述したファイルです。

## 概要

- このフォルダは、プロジェクト文書の上流資料を分類して管理を容易にする事を目的としています。
- [書類倉庫](../README.md)の下位フォルダとする事を、想定しています。(書類倉庫の整理用)
- 上流資料は、情報設計（IA）の前の工程での作成資料です。企画, 要求定義, 要件定義の各フェーズで作成されます。
  - 開発工程は、「開発工程」で記しています。
  - 当該文書は、「上流文書関係図」で記しています。
- ここでの上流資料は、以後のフェーズでの前提となる資料のサンプルです。実際の企画, 要求定義, 要件定義を経て作成されたものではありません。以後のフェーズの理解の為に、下流の実態から想像して作ったものです。
- この `README.md` ファイルは、書類の管理も兼ねているので、上流資料の新規作成, 移動, 削除の時は、必ず[一覧](#一覧)を更新して下さい。

## 一覧

- [開発工程](./development_process.md) - 一般的な開発工程と対応する文書一覧
- [上流文書関係図](./upstream_relationship.md) - 特に上流で重要な文書とその関係性
- [企画書](./project_proposal.md) - 目的・ターゲット・制約の洗い出し
- [要求定義書](./requirement_definition.md) - 発注側の希望・目的の明確化
