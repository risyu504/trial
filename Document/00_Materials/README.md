# 前提資料

この `README.md` ファイルは、このフォルダについての説明を記述したファイルです。

## 概要

- このフォルダは、プロジェクト文書の前提資料を分類して管理を容易にする事を目的としています。
- [書類倉庫](../README.md)の下位フォルダとする事を、想定しています。(書類倉庫の整理用)
- 前提資料は、開発・文書全体の枠組みを示す基礎資料です。前提や構成、語彙などを整理します。
  - プロジェクト文書の個別の配置は、**前提資料**の[文書体系・一覧](../00_Materials/document_structure.md)を参照。
- この `README.md` ファイルは、書類の管理も兼ねているので、前提資料の新規作成, 移動, 削除の時は、必ず[一覧](#一覧)を更新して下さい。

## 一覧

- [開発プロセス・概要](./development_process.md) - 一般的な開発プロセスをまとめたものです。
- [文書体系・一覧](./document_structure.md) - 開発に関する文書体系・一覧をまとめたものです。
- [文書関連図](./document_relationships.md) - 文書間の依存関係や関連図を記載。
- [使用ツール・言語方針](./tools_policy.md) - Sass / Pug の採用方針、gulp や VSCode の使用方針など、開発技術の前提条件を記載。
- [フォルダ構成・方針](./folder_policy.md) - フォルダ命名・分類に関する方針記録。
- [用語集](./glossary.md) - 特殊な語彙・略語の整理。現在は使用最小限の方針。
