# メンバの命名規則

このファイルは、開発の設計方針として、「メンバの命名規則」をまとめたものです。

このフォルダについての説明は、[`README.md`](./README.md) に記載しています。

## メンバ種別とスコープ別接頭語

1. メンバは、変数, 関数, mixinなどを想定しています。
2. 形式としては、ローワーキャメルケースを採用。厳密には接頭語の１, ２が英字小文字で、３が英字大文字英字小文字の組み合わせで以降がキャメルケースの名称であり、末尾２桁が２桁であるので、名称の部分はアッパーキャメルケースです。
3. スコープが mixin, function などのメンバ内の物を除き、名称はプロジェクトで一意になるようにする。(名称だけで、どこで定義されているか分かる事が目的です)
4. スコープで 関数/ブロック内のスコープは、命名では扱わない。ファイル内スコープとして表す。実質、グローバルか否かの区分けに留めるという事です。グローバルスコープ, ローカルスコープの２つです。
5. 例外として、引数は逆にプロジェクトで意味ごとに同一とする。尚、当該言語が用意している機能を意図した引数は、その機能と同じ引数名とする。Sassのビルトイン関数を意識しての規則です。
6. 例外として、キー項目とデータ項目からなるデータの項目名などは対象外とする。これは一般的な名称などのデータを、そのまま項目名とする事で索引の意味がある場合がある為です。
7. 例外として、クラス(ES6 以降のclass構文)は、慣習的な区別に合わせてアッパーキャメルケース(全ての単語の先頭を大文字)にする。
8. 上記を踏まえたうえで、名称は以下の形式とする。
    1. **先頭**: メンバのスコープを表します。言語によってスコープの判断は異なります。故に規約で意図を明示する事としています。
    2. **１桁目**: メンバの種類を表します。
    3. **２桁目**: データの型(用途, 戻り値, 出力するスタイル)を表します。

### 接頭語: スコープ（先頭）

|接頭語|意味|
|:-:|----|
|_|ローカルスコープを表す。|
|なし|グローバルスコープを表す。|

### 接頭語: メンバの種類（1桁目）

|接頭語|意味|
|:-:|----|
|l|ローカルスコープの変数。letでの定義を想定。|
|g|グローバルスコープの変数|
|c|定数。constはブロックスコープを持つので、グローバルスコープの場合や const がない言語では、varでの定義となります。|
|s|静的変数(static)。|
|f|関数。型が関数の変数もメンバの種類は関数として扱う。|
|m|mixin|
|a|引数(ローカルスコープですが`'_'`(アンダーバー)は付けません)|
|p|プロパティ(ローカルスコープですが`'_'`(アンダーバー)は付けません)|
|m|メソッド(ローカルスコープですが`'_'`(アンダーバー)は付けません)|
|v|任意, 未定|

### 接頭語: 型・戻り値・出力（2桁目）

| 接頭語 | 意味 |
|:-:|----|
|v|型が任意('any')である事を表します。戻り値, 出力がない場合も表します。"a"は引数で使用。|
|n|型が数('number')である事を表します。(「単位あり」も含みます)|
|u|型が数('number')で単位なし('unitless')である事を表します。|
|i|型が数('number')で整数 ('integer')のみを保持する事を表します。(「単位あり」は含みません)|
|s|型が文字列('string')である事を表します。(「引用符なし」も含みます)|
|c|型が色('color')である事を表します。文字列の色指定のみ保持での使用可|
|l|型がリスト('list'), 配列('Array')である事を表します。|
|m|[**廃止予定**]型がマップ('map')である事を表します。[JavaScriptにMapオブジェクトがあるので、o:オブジェクト('object')に統合]|
|o|型がマップ('map'), オブジェクト('object')である事を表します。|
|b|型が真偽値('boolean')である事を表します。|
|f|型が関数('function')である事を表します。ただし型が関数の変数はメンバの種類を関数として扱う。その場合はその関数の戻り値が2桁目となる。var fvTsFunction=()=>{}; |
|a|型が引数リスト('arglist')である事を表します。|
|x|型・戻り値・出力の表現が不要。mixinなどで使用。|

## 型/用途の接頭語

1. **３,４桁目**: フォルダ区分を表します。
     - 所属するフォルダのフォルダ区分を付けます。これにより、所属が分かり易くなります。
     - フォルダ区分はその開発環境で一意となるようにする。
     - 引数, 属性引数は業務で共通としたいので、所属ではなくて例外として、`'Ag'`, `'ag'`とします。
     - プロパティ, メソッドは業務で共通としたいので、所属ではなくて例外として、`'Ob'`とします。
     - 所属は役割を表す事がポイントで、例外を用意する場合でも、用途が不明となるような分け方はしないように注意願います。
2. **５,６桁目**: ファイル区分を表します。(フォルダ内でのみ一意であり、他のフォルダとの一意はフォルダ区分があるので考慮しない)
     - `'Cm'`: 一般(common)
     - `'Xx'`: 廃棄予定, 仮の記述
     - Xx: 英字大文字１文字から始まり、２文字目は英字小文字だけとします。必要に応じて各フォルダ内の内部規約に定めて使用します。
     - 引数, 属性引数, プロパティ, メソッドは例外として、ファイル区分を使用しません。
     - 上記以外: 名称を表します。５-８桁目を省略として９桁目以降の規約に準じます。
3. **７,８桁目**: ファイル内区分を表します。
     - nn: 数字だけとします。
     - ファイル区分を省略した場合は、指定は不可とします。
     - ファイル区分を指定した場合は、指定は任意とします。
     - ７桁目が英字大文字の場合、７桁目から名称と解釈します。
     - ７桁目が英字小文字の場合、５桁目から名称と解釈します。
4. **下２桁**: 用途を表します。
     - `'00'`: 内部処理用(非公開)
     - `'01'`: 基本処理
     - `'02'-'09'`: 基本処理の臨時版数管理
     - `'11'`: 一括処理
     - `'12'-'19'`: 一括処理の臨時版数管理
     - `'xx'`: 廃棄予定(既存のものに付与して、仮の削除とする。)
     - 省略可能です。(名称の末尾に制限が付きます)
5. **９桁目以降(下２桁除く)**: 名称を表します。
     - Xxxxx: 英字大文字１文字から始まり、２文字目は英字小文字, それ以降は０文字以上の英数字小文字だけとします。(キャメルケース)(一部区切りの頭を英字大文字にします)
     - 検索を容易にする為に、短すぎる名称は避ける。
     - 記述を容易にする為に、長すぎる名称は避ける。
     - ファイル区分を省略した場合は、ファイル区分との区別の為に最初の単語を３文字以上とする。
       - ７桁目が英字大文字の場合、７桁目から名称と解釈します。
       - ７桁目が英字小文字の場合、５桁目から名称と解釈します。
       - ７桁目が数字の場合、５桁目から名称と解釈します。
     - 下２桁を省略した場合に末尾を数字とする場合は、１桁だけを可能とします。(末尾２桁以上の数字は、その下２桁を用途と判断します)
     - ８桁目以前で省略があった場合に、桁位置がずれるので注意
