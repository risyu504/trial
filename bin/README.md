# 実行ファイルフォルダ readme

この `README.md` ファイルは、このフォルダに関しての説明を記述したファイルです。

## 概要

- このフォルダは、**プログラムの実行ファイル**を収めるフォルダです。
- [開発フォルダ](../README.md)の下位フォルダとする事を、想定しています。
  - (フォルダ構成は「[フォルダ構成・方針](../Document/00_Materials/folder_policy.md)」で記しています。)
- 名称: **Bin**, 区分: **Bn** です。

## 使い方

- 意味別にフォルダを用意します。
- 各ツールで実行します。実行方法は、ツールの使い方の方で調べて下さい。

## 内部規約

1. 主に [コーディングルール](../Document/20_Design/codingrules.md) に準じます。
2. 外部データを考慮して、ある程度の例外を認めます。ただ出来るならば上記のコーディングルールに合わせるように努力する事を求めます。尚、内部で作成した実行ファイルなどは規約に準じる必要があります。
3. このフォルダのフォルダ区分は、'**Bn**'(Bin) です。
4. このフォルダのファイルに関する規約は、以下の通りとします。
    1. このフォルダのファイル区分は、以下の通りとします。
        - 現在はありません。
    2. ファイル名は、外部データを考慮して自由とします。
        1. 内部で作成した実行ファイルなどは規約に準じる必要があります。
        2. 内部で作成した実行ファイルでも、外部データなどとの兼ね合いで上記のコーディングルールに合わせるのが困難な場合は、自由とします。つまりここでのコーディングルールは厳守ではなく推奨, 努力目標程度のものです。
        3. 外部から入手した実行ファイルのファイル名を無理して命名規則に合わせて変える事は、入手元を分かり難くするので、お薦めしません。
    3. ファイルは必要に応じて、フォルダで分ける事とします。
