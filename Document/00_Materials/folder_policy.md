# フォルダ構成・方針

このファイルは、開発の前提資料として、「フォルダ構成・方針」をまとめたものです。

このフォルダについての説明は、[`README.md`](./README.md) に記載しています。

## かんたんな説明

この文書には、「どのようなフォルダ構成で開発するか」を定めたルールが書かれています。

## フォルダ構成・方針とは

- 本文書は、フォルダの命名や分類に関する方針を記録するものです。
- 文中で示すフォルダ名は説明のための例示であり、実際に使用する際は命名規則に従って適切に命名し直します。

### フォルダの構成例(アルファベット順)

```一般
.
├── assets/
├── bin/
├── config/
├── data/
├── dest/
├── dist/
├── docs/
├── module/
├── node_modules/
└── src/
```

## フォルダの一般的な分類

- **ソースコード関連**: `src/`, `module/`
- **自動化・ビルド**: `gulpfile/`
- **ビルド成果物**: `dest/`, `dist/`
- **データ**: `data/`, `assets/`
- **設定**: `config/`
- **ドキュメント**: `docs/`
- **補助・その他**: `node_modules/`, `bin/`

## 各フォルダの役割と補足

### src/

- Sass, Pug, TypeScript など、エントリーポイントとなるソースコードを格納します。プロジェクトの出発点となるフォルダです。

### module/

- FLOCSSベースの構造に基づき、開発資源を整理します。  
`package` ではなく `module` を用いるのは、Node.jsの `package.json` との混同を避けるためです。

### gulpfile/

- Gulp用の処理を格納するフォルダです。  
gulpfile.js の簡素化と、構成管理の目的でフォルダ化しています。
意図的に gulpfile.js に近い名称としています。

### dest/

- 成果物の出力用フォルダです。

### dist/

- 成果物の配布用フォルダです。

### data/

- データ専用のフォルダです。

### assets/

- 静的資源などを格納するフォルダです。

### config/

- 設定ファイル用のフォルダです。

### docs/

- ドキュメント用フォルダです。ただし、ツールによっては生成先が `docs/` に固定されている場合があるため、安全性を考慮する場合は `document/` の使用を推奨します。

### document/

- 安全性を考慮して `docs/` の代替として使用するドキュメント用フォルダです。

### node_modules/

- Node.jsのパッケージ管理フォルダです。

### bin/

- 実行ファイル用フォルダです。

## 方針

### 全体方針

- 「製造」と「検証・配布」の分離を意識します（※単体テストは製造に含む）。
- Node.js（Gulp）の使用により、生成・命名変更・複写・移動を自動化できるため、事前に構成を過度に厳密化する必要はなく、需要に応じた柔軟な運用が可能です。
- 一般的な名称を採用し、必要に応じて命名規則に従って調整します。

### src/ の方針

- 対象言語別にディレクトリを構成します。
- 各言語ごとの下位構成は、実運用を意識して設計します。

### module/ の方針

- CSS設計である FLOCSS の構造を参考に構成します。
  - FLOCSSについては以下を参照: [https://github.com/hiloki/flocss](https://github.com/hiloki/flocss)

### gulpfile/ の方針

- CSS設計である FLOCSS を参考に構成します。
- JavaScript 雛型ベースの流用を前提とした構成とします。
- `gulpfile/` と `module/` の共通処理における JavaScript のメンバ構成は、できる限り同一となるように努力します。

### dest/ の方針

- 配布用構成を意識し、単体テストが可能な粒度を目指します。

### dist/ の方針

- 使用しません。  
配布は、総合テストが可能な別環境から行う方針です。

### config/ の方針

- 下位フォルダによる分類は行いません。
- 設定ファイルは、可能な限りここに集約します。

### document/ の方針

- ツールによっては生成先の既定値が `docs/` に固定されている場合があるため、安全性を優先して `document/` を採用します。
- 構成方針は「文書体系・一覧」文書に準拠します。
