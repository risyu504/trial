# 文書関連図

このファイルは、開発の前提資料として、「プロジェクト内の各文書がどのように関連し合っているか」を図式的に整理したものです。

このフォルダについての説明は、[`README.md`](./README.md) に記載しています。

## この文書について

この文書は、プロジェクト内の各文書がどのように関連し合っているかを図式的に整理したものです。  
主に「依存関係」や「参照関係」「グループ構造」を明示し、保守・参照時の負担軽減を目的とします。

## 概要図（文書グループ単位）

### 作業工程・管理文書グループ

- 「開発プロセス」や「文書の整理・管理」に関する、メタ的・上位の管理文書群

```概要
[開発プロセス・概要] → [文書体系・一覧]
                   ↘ [文書関連図]
```

### 構造設計・技術基盤グループ

- プロジェクトの技術的な選定方針と、**構造設計（ディレクトリ構成）** の方針を示すグループ

```概要
[基本原則] ─┬─[使用ツール・言語方針]
           └─[フォルダ構成・方針]
```

### 構造設計

```概要
[構造設計]──┬─[関数・mixin 設計]
           ├─[ユーティリティ設計]
           ├─[HTML構成設計]
           └─[定数・外部データ設計]
```

### 規約ルール・命名規則グループ

- 命名や記述スタイルなど、**コード内部の規律・統一性**に関するルール群

```概要
[基本原則]──┬─[補足・内部規則構造図]
           ├─[フォルダ／ファイル命名規則]
           ├─[メンバ命名規則]
           └─[CSSのセレクタ命名規則]
```

### 実装

```概要
[実装ガイド] → [記述規則] → [コメント記述規則]

[README.md]（各フォルダ）→ [フォルダ構成・方針]
```

※矢印は「参照」または「構成依存」を表す
