# 使用ツール・言語方針

このファイルは、開発の前提資料として、「使用ツール・言語方針」をまとめたものです。

このフォルダについての説明は、[`README.md`](./README.md) に記載しています。

## かんたんな説明

ここに書いてあるのは、「どんな道具や言葉を使って開発するか」をまとめたルールです。

たとえば、

- 「この作業って、何を使えばいいの？」
- 「なんでこのツールを使ってるの？」
- 「他のやり方とどうちがうの？」

そんなことを考えるときのヒントになるように書いてあります。

むずかしく見えるかもしれないけど、自分があとで読みかえしても分かるように、なるべくていねいに書いてあります。ひとつずつ読めば大丈夫なので、安心してください。

## 使用ツール・言語方針とは

この文書では、プロジェクトで使用するプログラミング言語や開発ツール（たとえば JavaScript, Sass, Pug, Gulp, VSCode など）について、

- **どのツールや言語を使っているか**
- **どのような使い方を前提としているか**
- **なぜその選択をしているのか**

といった「使用方針」をまとめています。

目的は、将来の自分が「技術的な前提」や「選定理由」をすばやく把握できるようにすることです。操作マニュアルではなく、あくまで**考え方や基準を明らかにするための資料**です。

特定の手順や設定方法よりも、「選定・使用の方針」を重視して記録しています。

## 採用言語

### JavaScript

JavaScript を使用する。TypeScript は、JavaScript のコメントなどの検証手段として限定的に用いる。

JavaScript は、Node.js（Gulp）での開発用ツールとしての使用と、HTML 上での動作を目的とした実装の両面がある。ただし、環境の違いにかかわらず、同一の設計思想で統一して記述する。

### Sass

CSS の作成には Sass を用いる。ビルドには Gulp を使い、VSCode の拡張機能は使用しない。構文は SCSS を採用する。

Sass の採用理由は、セレクタの管理や更新を容易にするためである。SCSS 構文を選んだのは主に慣れによるものだが、CSS 出力の整形を視野に入れると、インデントに縛られる SASS 構文は柔軟性に欠けると判断した。

なお、出力の見た目にこだわらなければ、SASS 構文の方が簡素で書きやすいという選択もある。ただし、そのぶん文字列検索やパターン抽出の面では不便である。

### Pug

HTML の作成には Pug を用いる。ビルドには Gulp を使用し、VSCode の拡張機能は使用しない。

Pug は、CSS におけるセレクタの管理・更新との連携を重視して採用しており、セレクタ編集機能はテンプレート構造から明確に独立させている。

記事データの外部 JSON 化は、Pug が外部データを読み込めるという副次的な利点を活用したものであり、言語選定の主目的ではない。外部データが使えない場合は、内部定数として保持する方針とする。

外部データの活用により、Web サイトのページ間関係の自動管理（ナビゲーションなどの自動生成）が可能になっている。

さらに副産物として、コードを外部データ（JSON など）として管理し、HTML 文書として出力する処理と、HTML コードとして表示するためのエスケープ処理を分けることで、文書と表示用コードの同一性を高める運用が可能となっている。

### markdown

文書作成には Markdown を使用する。

主な採用理由は、リンク機能の便利さにある。また、GitHub への投稿も視野に入れている。

文書は HTML に変換して Web サイトに掲載することで、学習・参照用としての活用も想定しているため、Word や Excel を使った高度な装飾は不要と判断している。

## 採用ツール

### VSCode

テキストエディタとして VSCode を採用する。Sass や Pug のビルドには使用しない。

### Node.js(Gulp)

Sass や Pug のビルドのために、Node.js 上で Gulp パッケージを使用する。

Gulp 採用の主な理由は、Sass / Pug の一括ビルドが容易なこと。また、外部データ（JSON 等）の編集処理も Gulp に組み込むためである。

### JSDoc, SassDoc

JSDoc および SassDoc の文書生成にも Gulp のパッケージを用いる。

これらはコメントを元にドキュメントを生成するツールであるが、主にコメントの記述・検証を補助する目的で使用している。生成された文書を直接的に実用とする意図はないが、補助資料としての利用はあり得る。ただし、その利便性の検証は行わない。
