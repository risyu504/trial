# 書類倉庫

このフォルダは、書類を収める為のものです。

## 概要

- このフォルダは、文書ファイルを[ルートフォルダ](../README.md)に置くと煩雑なので、書類をひとつにまとめてルートフォルダを簡素にする事を目的としています。
  - フォルダ構成は「[フォルダ構成・方針](../Document/00_Materials/folder_policy.md)」で記しています。
- プロジェクト文書の個別の配置は、**前提資料**の[文書体系・一覧](./00_Materials/document_structure.md)を参照。

## 書類分類

- [上流資料](./000_Upstream/README.md) - プロジェクト文書の上流資料。
- [前提資料](./00_Materials/README.md) - 開発・文書全体の枠組みを示す基礎資料。前提や構成、語彙などを整理。
- [基本原則](./10_Principles/README.md) - 設計判断の最上位規範。「設計の設計」に相当。
- [設計方針](./20_Design/README.md) - 命名・構造・記述に関する各種設計指針。機能や記法のルールも含む。
- [内部仕様](./30_Usagerules/README.md) - コード・変数のローカル仕様。BEMや制御変数など内部向け技術文書。
- [開発環境](./40_Reference/README.md) - 環境構築・実行基盤の手順やメモ。OS、Node.js、Gulp等の記録。
- [補助資料](./50_Supporting/README.md) - チェックリストや調査メモ、清書前の技術メモなど。
